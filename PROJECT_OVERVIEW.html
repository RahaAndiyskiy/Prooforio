Prooforio

Краткий обзор проекта

Prooforio — это сервис для сбора отзывов по персональной ссылке и последующего экспорта отзывов в виде готовых карточек для шаринга.

Что умеет проект

  • Регистрация и вход пользователя.
  • Публичная страница для сбора отзывов по ссылке /review/[username].
  • Панель пользователя с отзывами и ссылкой для распространения.
  • Страница шаринга отзыва с выбором шаблона и живым preview.
  • Экспорт карточек в PNG и JPEG.

Ключевая идея

  • Отзыв сначала собирается как данные.
  • Потом данные нормализуются в общий шаблонный контекст.
  • Дальше один и тот же отзыв можно показать в разных визуальных шаблонах.
  • Preview и экспорт используют общий реестр шаблонов.

Основные маршруты

  • / — главная страница.
  • /login — вход.
  • /register — регистрация.
  • /dashboard — панель пользователя.
  • /review/[username] — публичная форма отзыва.
  • /share/review/[reviewId] — выбор шаблона и экспорт карточки.
  • /api/reviews — создание отзыва.
  • /api/share/review-image — генерация изображения карточки.

Основной стек

  • Next.js App Router
  • React + TypeScript
  • Tailwind CSS
  • Supabase для данных и auth
  • satori + @resvg/resvg-js + sharp для экспорта изображений

Структура проекта

src/ app/ Next.js routes и API routes entities/ доменные типы profile/review features/ auth/ логика входа и регистрации dashboard/ панель пользователя review/ форма отзыва и список отзывов share-review/ шаблоны, preview и экспорт карточек shared/ api/ доступ к profile/review данным lib/ общие утилиты, Supabase client ui/ кнопки, карточки, input и другие примитивы widgets/ header/ общий header проекта

Как устроен экспорт карточек

  • Отзыв передается в шаблонный контекст: brand / header / rating / body / footer.
  • Каждый шаблон описан как отдельный компонент.
  • Выбранный шаблон сначала рендерится в SVG через satori.
  • Потом SVG конвертируется в PNG через Resvg.
  • Для JPEG используется дополнительная конвертация через sharp.

Текущее состояние

  • Базовый flow проекта уже работает end-to-end.
  • Система шаблонов подготовлена к росту: есть metadata, фильтры и основа под разные форматы.
  • Следующий большой этап — визуальная система шаблонов и расширение библиотеки карточек.
