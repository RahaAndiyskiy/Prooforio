Style Builder Concept

Это не новый отдельный проект и не Canva-клон. Это будущая надстройка над уже существующей системой шаблонов, которая позволит пользователю собирать собственный стиль карточки по шагам, не переписывая текущий preview/export pipeline.

Суть идеи

Пользователь берет уже существующий отзыв и не выбирает только готовый шаблон, а проходит пошаговый конструктор, где настраивает внешний вид карточки: формат, скругления, цвета, шрифты, композицию, расположение рейтинга, текста и декоративных элементов.

Что это не должно быть

  • Не отдельная новая платформа с нуля.
  • Не свободный canvas-редактор с бесконечным drag and drop.
  • Не замена текущему экспортному движку.
  • Не полная перезапись текущих шаблонов.

Что это должно быть

  • Ограниченный пошаговый конструктор.
  • Надстройка над текущими шаблонами и preview.
  • Система параметров, а не ручного рисования.
  • Управляемая визуальная сборка карточки из готовых блоков.

Как это должно строиться поверх текущего проекта

Отзыв -> нормализованный контент (brand / header / rating / body / footer) -> style config пользователя -> layout rules -> primitive blocks -> preview -> export PNG / JPEG

То есть новый слой добавляется между данными отзыва и финальным шаблоном. Существующий экспортный pipeline остается, но вместо жестко написанного шаблона он сможет использовать параметры, выбранные пользователем.

Какие сущности понадобятся

  1. Style Config — объект с настройками внешнего вида.
  2. Layout Schema — правила расположения блоков.
  3. Primitives — переиспользуемые элементы карточки.
  4. Builder Steps — пошаговый UI настройки.

Какие параметры логично настраивать

  1. Формат и ориентация.
  2. Скругления и рамки.
  3. Цвет фона и цвет карточки.
  4. Шрифт, размер и насыщенность текста.
  5. Положение текста, автора и рейтинга.
  6. Декоративные элементы и акцентные зоны.

Пример будущего flow пользователя

  1. Выбрать отзыв.
  2. Выбрать формат: landscape, square или story.
  3. Выбрать базовый стиль или стартовый пресет.
  4. Настроить фон, цвета, углы и типографику.
  5. Настроить композицию текста, рейтинга и подписи.
  6. Посмотреть live preview.
  7. Сохранить стиль как preset или сразу экспортировать карточку.

Почему это не потребует переписывать проект заново

  • Уже есть нормализованный контент карточки.
  • Уже есть реестр шаблонов и metadata.
  • Уже есть подготовка под форматы и фильтры.
  • Уже есть единый preview/export pipeline.
  • Новый builder можно строить как верхний слой над текущей архитектурой.

Минимальная архитектурная цель на будущее

Дойти до состояния, где шаблон — это не только вручную написанный компонент, а комбинация из данных отзыва, style config и layout schema. Тогда готовые шаблоны, пользовательские стили и будущий конструктор смогут жить в одной системе.
